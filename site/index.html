The world's first Rust-native LLM inference server for Apple Silicon.

Single static binary. Zero GC. Continuous batching, paged KV cache, fused Metal kernels, MLX-Int4, dynamic quantization, prefix cache, speculative decoding. OpenAI- and Anthropic-compatible. Triple transport: HTTP, Unix socket, gRPC.

What it does

Built for serving real workloads, not chatbot demos.

Most LLM runtimes on Mac (Ollama, mlx-lm, llama.cpp) optimize for one developer, one chat. Velox is built for the next thing: multi-tenant, multi-model, low-latency, OpenAI-compatible serving on a Mac Studio under your desk.

⚡

Continuous batching

New requests join the in-flight batch every step. Per-token cost stays flat as concurrency grows — 1 user and 16 users share the GPU efficiently.

▦

Paged KV cache

vLLM-style page allocator (16-token pages). Packs 2-4× more concurrent requests in the same VRAM than naive max-seq allocation.

▣

Fused Metal kernels

Custom MSL kernels for paged decode/prefill attention, batched RoPE, batched scatter, and qmm_4bit dequant. Compiled into the binary.

⌘

Prefix cache

Page-level KV cache deduplication for shared system prompts. Measured 6.5× speedup on repeated long-context conversations.

≡

MLX-Int4 + dynamic quant

Run mlx-community 4-bit checkpoints natively, or dynamically quantize any BF16 model at load time. 4× memory reduction over BF16.

⇄

Triple transport

HTTP (OpenAI + Anthropic compat), Unix domain socket (~30µs faster locally), and gRPC with HTTP/2 streaming for typed clients. All same port, same scheduler.

Install

One command. Apple Silicon required.

Quick install (recommended) 
curl -fsSL https://velox.soflution.com/install.sh | sh

velox serve --model-dir ~/.velox/models
Homebrew tap 
brew tap soflutionltd/tap \
  https://github.com/Soflutionltd/velox

brew install velox
Build from source 
git clone https://github.com/Soflutionltd/velox && cd velox
brew install protobuf
cargo install --path . --features candle-metal

Requirements

  • → macOS 14+ on Apple Silicon (M1, M2, M3, M4)
  • → 8 GB unified memory minimum (16 GB recommended for 7B+ models)
  • → Rust 1.78+ and protoc (for building from source)
